Mandatory course materials

Books

  • Goetz et al: Java Concurrency in Practice, Addison-Wesley 2006. The Academic Bookstore will sell it at ITU at start of term. Also at Amazon.co.uk. Example code at http://jcip.net/.
  • Herlihy and Shavit: The Art of Multiprocessor Programming. Revised first edition, Morgan Kaufmann 2012. Chapters 3 (safety and liveness concepts) and 18.1, 18.2 (compositionality, transactional memory). At Amazon.co.uk
  • Sestoft: Java Precisely 3rd edition (MIT Press 2016, draft), sections 11.13, 11.14, 23, 24, 25. PDF available from LearnIt.

Articles

  • Sestoft: Microbenchmarks in Java and C#. Free at http://www.itu.dk/people/sestoft/papers/benchmarking.pdf with the corresponding Java example code.
  • Harris, Marlow, Peyton-Jones, Herlihy: Composable memory transactions. Communications of the ACM 51, 8 (August 2008) 91-100; PDF available from LearnIT.
  • Cascaval et al: Software Transactional Memory: Why Is It Only a Research Toy? ACM Queue 2008. Free at http://queue.acm.org/detail.cfm?id=1454466
  • Michael and Scott: Simple, Fast, and Practical Non-Blocking and Blocking Concurrent Queue Algorithms. PODC 1996. Free at http://www.research.ibm.com/people/m/michael/podc-1996.pdf
  • Chase and Lev: Dynamic circular work- stealing deque, SPAA'05, 2005, sections 1, 2 and 5; see also LearnIT.
  • Armstrong, Virding, Williams: Concurrent Programming in Erlang. Part 1 free at http://www.erlang.org/download/erlang-book-part1.pdf.

Other

  • The Multiverse Software Transactional Memory library for Java: Multiverse API Javadoc (local copy) and the multiverse-core-0.7.0.jar (library implementation).

Optional course materials

  • Bloch: Effective Java. Third edition, Addison-Wesley 2018. Item 17 on immutability and items 78-84 on concurrency. At Amazon.co.uk. Contains much other wisdom for Java (and .NET) developers; you should own this book and use it.
  • Sestoft: Java Precisely. Second edition, MIT Press 2005. Especially chapters 9 and 10 on classes, inner classes and anonymous classes, and chapter 16 on threads. At Amazon.co.uk
  • Sutter: The free lunch is over. Free at http://www.gotw.ca/publications/concurrency-ddj.htm
  • If you want a deep understanding of how modern multicore cache and memory hardware design affects concurrency performance, these seem to be good overviews, although far too detailed for this course: McKenney: Memory barriers, a hardware view for software hackers (IBM, 2010) and Drepper: What every programmer should know about memory (Redhat, 2007).
  • Budd, Lipton, Sayward, DeMillo: The design of a prototype mutation system for software testing, 1978. Free at http://www.computer.org/csdl/proceedings/afips/1978/5086/00/50860623.pdf.
  • On transactional memory: Peyton Jones: Beautiful concurrency, 2007; and Larus and Kozyrakis: Transactional memory, 2008 (free access within ITU); and Dan Grossman's very enlightening analogy between transactional memory and garbage collection; and hardware support via the Intel Transaction Synchronization Extensions (2012) and also pages 8-1 through 8-24 of the Instruction Set Extensions Programming Reference; Robert Ennals: Software transactional memory should not be obstruction-free, 2006, PDF; and a response to Cascaval's critique of STM by Dragojevic et al: Why STM can be more than a research toy CACM, April 2011. (PDF accessible for free from within ITU's network).
  • On memory models: see Java Language Specification below; and Ostrovsky: The C# Memory Model in Theory and Practice, in MSDN Magazine, part 1 (December 2012) and part 2 (January 2013); and The C# Language Specification (standard Ecma-334), especially section 10.10 and 17.4.3, and Joseph Albahari's Threading in C# Part 4: Advanced Threading.
  • Akka+Java Actor Snapshot
  • The Akka+Java documentation
  • Video Talk on Akka by Viktor Klang (on YouTube)
  • Video Talk on Akka by Jonas Boner (on YouTube)
    • Slides for the Video Talk on Akka (from GOTO conference, Aarhus, Denmark).

Even more optional materials

  • Nancy Leveson's 1995 report on the Therac-25 accidents, caused by incompetent concurrent programming.
  • Per Brinch Hansen on Java's insecure parallelism, 1999.
  • Hoare's paper on monitors, CACM 17, 10 (October 1974) 549-557.
  • The CERT Secure Coding Standard for Java especially sections 09 on Locking, 10-12 on Threads, and 08 on Visibility and Atomicity.
  • McKenney (ed.): Is Parallel Programming Hard, And, If So, What Can You Do About It? IBM 10 March 2014. Free at https://www.kernel.org/pub/linux/kernel/people/paulmck/perfbook/perfbook.html. Chapters 2 and 3.
  • A good book about .NET parallel programming is Campbell, Johnson, Miller, Toub: Parallel Programming with Microsoft .NET, Design Patterns for Decomposition and Coordination on Multicore Architectures. Microsoft Press 2010. There is an online version and also a printed version at Amazon.com.
  • M. Herlihy: Wait-free synchronization. ACM Toplas 11, 1 (January 1991) 124-149. Free PDF at Herlihy's homepage.
  • N. Shavit: Data structures in the multicore age. Communications of the ACM 54, 3 (March 2011) 76-84. Free inside ITU from the ACM Digital Library.

Resources

  • Java concurrency tutorial. Free at http://docs.oracle.com/javase/tutorial/essential/concurrency/.
  • Java streams tutorial, parallelism section. Free at http://docs.oracle.com/javase/tutorial/collections/streams/parallelism.html.
  • Java Swing GUI Toolkit tutorial, concurrency lesson at http://docs.oracle.com/javase/tutorial/uiswing/concurrency/index.html.
  • Java 8 class library documentation online.
  • Java 8 development kit download
  • Java 8 language specification in PDF and online as HTML; especially 17.1 synchronization and 8.3.1.4 volatile fields and 17.5 final field semantics and the somewhat complicated 17.4 memory model.
  • Brian Goetz about the design of the Java 1.5 memory model: part 1 and part 2, IBM 2004.
  • JSR 305 thread-safety annotations jar file for @GuardedBy and similar. Download the jar file, put it someplace such as ~/lib/jsr305-3.0.0.jar, add import javax.annotation.concurrent.GuardedBy; to your Java source files, and mention the jar file on the classpath when compiling Java files, as in javac -cp ~/lib/jsr305-3.0.0.jar *.java
  • Akka homepage. http://akka.io/ (Documentation, FAQ, Download, ...).
  • Akka+Java API
